The method of providing feedback on OSCIA's website was regarded as dated and non-intuitive.

When clicking the feedback button, the user's mail client is opened with the CC field prepopulated. While this does work, it lacks a visually appealing UI and there are no guidelines to submitting feedback, resulting in very vauge messages.


I was tasked with redesigning the feedback process, with the goal of providing a clean, inutuivie UI while simultanious increasing the amount of usable data collected.


To implement these changes a submittable form was chosen. This provdes advtnages as the UI can be customized rather than using the user's email client, and additional data can be transfered in the background.

The feedback form can be seen in an image on the right side of this page.

To improve accuracy and depth of the data collected, there are extra fields such as type of feedback, name, email, and more

To further improve the value of the feedback, data is collected in the background silenty. This involves IP address, current URL, web browser, OS, exact time, and more.

Overall, the new format of submitting feedback was a great success. It provides a smoother user experince, guides the user into submitting complete feedback through input validation, and the increased data collection allowed the IT team to parse through spam emails much easier
