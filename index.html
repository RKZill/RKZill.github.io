For one of my first projects at OSCIA, I was tasked with the creation of an improved feedback process

The old feedback method involved the user clicking 'feedback', which opened the user's mail client and prepopulated the 'To' field. This works for it's purpose but could be improved.

To improve this process, there was two key questions that needed to be looked at:

  1. The user's experince; How can I design a visually appealing UI that involves an easy to use process?
  2. The data collected; How can we gain more useful data through user submitted feedback, when the user can submit as little as they want?

To improve user experince, a pop-up with a form was chosen.

The form fields include validation which highlights the error field and dispalys a short error message.

To improve the data collected, a multitude of hidden data is collected to help further narrow down potential issues. This includes:

  1. User IP address. This has proved useful in real applications, where our IT was able to use this tool to identify spam feedback coming from outside the country
  2. User web browser. Knowing what browser the feedback came from can assist in the idenfication of browser specific issues.
  3. Current URL where feedback has been submitted, for percisely knowing where exactly the issue was found
  4. And much more!

Overall, user's do not submit a great amount of feedback, but this tool has helped improved the user experince when submitting feedback, and greatly improved the usibility of feedback that is submitted
